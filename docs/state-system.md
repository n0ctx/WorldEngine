# 状态系统

本文说明 WorldEngine 的状态字段如何定义、如何在会话中运行、如何自动更新，以及如何与提示词条目联动触发叙事变化。

---

## 设计目标

状态系统解决的问题：AI 在多轮对话中很难自主维护"角色当前的情绪、好感度、体力"等运行时状态，用户也不应该每轮手动在提示词里重申"当前好感度是 60 分"。

WorldEngine 的做法是：
1. 把状态字段定义在配置层（世界 / 角色 / Persona）
2. 每个会话开始时复制一份运行时快照，各自独立变化
3. 每轮 AI 回复后，由后台 LLM 自动解析对话并更新状态值
4. 状态值通过提示词 [5][6][7] 段注入上下文，AI 每轮都能看到当前状态
5. 状态值可以触发对应的提示词条目（`state` 类型），让叙事氛围随状态自动切换

---

## 三层状态

| 状态层 | 挂载对象 | 典型字段 |
|---|---|---|
| 世界状态 | 世界（所有会话共用模板，快照独立） | 政局稳定性、戒严等级、当前季节 |
| 角色状态 | 角色 | 好感度、体力、当前情绪、持有物品 |
| 玩家状态 | Persona | 声望、金币、已解锁路线 |

三层状态分别由 `renderWorldState` / `renderPersonaState` / `renderCharacterState` 渲染为文本，注入提示词 [5][6][7] 段。

---

## 字段类型

| 类型 | 示例值 | 说明 |
|---|---|---|
| `text` | `"平静"` | 自由文本 |
| `number` | `75` | 数值，可设置 min / max |
| `boolean` | `true / false` | 开关 |
| `enum` | `"友好"` | 从预定义选项中选一个 |
| `list` | `["火把", "短剑"]` | 列表，AI 可追加/删除项目 |

每种类型有全局默认值（在世界/角色配置页设置）。会话开始时，默认值复制成运行时值，之后独立变化，不影响其他会话和全局默认值。

---

## 会话隔离

同一个角色在不同会话里可以处于不同的状态：

```
角色"艾莉娅"的默认状态：好感度=50

会话 A：通过对话好感度提升到 80，状态快照 → 好感度=80
会话 B：剧情走向不同，好感度下降到 30，状态快照 → 好感度=30

角色配置页的默认值仍然是 50
```

状态快照存储在 `session_states` 表，每个字段独立一行，关联 `session_id`。

## 更新模式

每个世界/角色/Persona 可以独立设置状态更新模式：

### `llm_auto`（自动）

每轮 AI 回复完成后，后台副模型（aux_llm）分析本轮对话内容，按字段类型生成更新指令并写入数据库。更新异步执行，不阻塞对话流程（优先级 2，不可丢弃）。

AI 更新状态时遵循以下规则：
- `number` 类型：给出增减量或新值
- `list` 类型：给出追加项或删除项，不替换整个列表
- 更新前先读取当前值，确保在合理范围内变化

### `manual`（手动）

AI 不自动更新，只有用户在状态面板手动编辑时才会改变。适合不想让 AI 擅自修改状态的场景，或者需要精确控制状态演进的 GM 风格玩法。

## 状态触发提示词条目

`state` 类型的提示词条目在满足条件时自动激活注入。条件使用表达式语法：

```
好感度 > 80          # 数值比较
当前情绪 == "愤怒"   # 枚举/文本匹配
已开战 == true       # 布尔值
持有物品 includes "古代地图"   # 列表包含
```

多个条件之间为 AND 逻辑，全部满足才命中。

**典型用法**：

| 条件 | 触发的条目内容 |
|---|---|
| 好感度 > 80 | 角色对玩家态度亲密，主动分享秘密 |
| 好感度 < 20 | 角色对玩家保持警惕，语气冷漠 |
| 戒严等级 == "高" | 城市街道有士兵巡逻，玩家需要通行证 |
| 持有物品 includes "魔法石" | 角色察觉到玩家身上的魔力波动 |

## 状态回滚

编辑历史消息、删除消息或重新生成（regenerate）时，会话状态需要还原到截断点之前的值。

WorldEngine 通过两种方式实现状态回滚：

### Turn Record 快照

`turn_records` 表（每条 turn record）不直接存储状态值，而是关联当时已写入 `session_states` 的 round 信息。截断 turn record 后，`session_states` 中对应 round 之后的状态更新被删除，状态自动还原到截断点的值。

### 并发屏障

状态更新属于优先级 2 的不可丢弃任务。所有触发状态回滚的路径（regenerate / 编辑 / 删除）都先等待 `waitForQueueIdle(sessionId)`，确保当前轮次的状态更新任务完成（或取消），再执行截断，避免回滚后又被旧任务覆写。

## 关键文件

| 文件 | 职责 |
|---|---|
| `backend/memory/combined-state-updater.js` | 状态更新推理（`updateAllStates`），列表字段裁剪 |
| `backend/memory/recall.js` | `renderWorldState` / `renderPersonaState` / `renderCharacterState` 渲染状态为文本 |
| `backend/db/queries/session-states.js` | 状态快照的 CRUD 查询 |
| `backend/db/queries/entry-conditions.js` | 提示词条目的 state 条件读取 |
| `backend/prompts/entry-matcher.js` | state 类型条目的条件判断（AND 逻辑） |
