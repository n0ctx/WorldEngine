# History References

历史决策、迁移背景与已踩坑记录入口。

## 什么时候读

- 需要理解为什么代码或文档现在长这样
- 需要查兼容约束、已踩过的坑

本主轴承接的高频跨模块任务片段：

- 跨模块任务里需要补历史背景、迁移原因或旧路径下线时间
- 无法确定某个“奇怪约束”是否是历史兼容遗留时

## 先读哪几页

1. [`changelog.md`](changelog.md)：按时间记录的迁移、踩坑与回归说明

## 高频任务快速分流

- 想知道某次重构为何存在：搜 `changelog.md`
- 想确认旧路径或旧结构何时下线：搜 `changelog.md`

## 真源与非真源

- 真源：历史决策、兼容背景、迁移提醒
- 非真源：当前运行时行为与数据库字段

## 何时同步

- 新增兼容约束、迁移提醒、需要未来 agent 记住的坑点时
