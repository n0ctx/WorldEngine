## 快速规则

1. **所有任务**先读 §0-§2（文档边界 + 目录结构），确认文件落点和副作用规则。
2. **不改后端行为的前端任务**（如纯 UI 调整）：读完 §0-§2 + §12 即可，通常不需要深入 §3-§11。
3. **不改前端行为的后端任务**（如改 memory、改 state）：读完 §0-§2 + 对应章节即可，通常不需要读 §12。
4. **跨前后端的任务**（如新增接口）：§0-§2 + §14.1 + §12。
5. **改数据库**：直接读 SCHEMA.md，不需要读 ARCHITECTURE.md 记忆相关章节。
