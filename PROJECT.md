# WorldEngine · 产品形态规划

> 本文件是产品级规划文档，描述用户将看到什么、能做什么、体验是什么。
> 技术实现细节见 `ARCHITECTURE.md`，数据结构见 `SCHEMA.md`，执行约束见 `CLAUDE.md`。
> 最后更新：2026-04-22

---

## §0 产品定位

**WorldEngine 是一个 AI 辅助的世界构建与沉浸叙事工具。**

用户不只是在"和角色聊天"——他们在**建造一个活的世界，然后在里面写故事**。世界有历史、有地点、有势力、有关系网络；故事里发生的一切都在改变这个世界的状态；状态的变化反过来驱动下一段叙事。

核心用户画像：
- 严肃的创意写作者和互动小说爱好者
- 有自己世界观设定的 TRPG 玩家/GM
- 想用 AI 辅助建立复杂角色关系与世界背景的人

不面向的用户：只想快速开一个对话和角色聊天的人（SillyTavern 更适合他们）。

## §1 整体结构重构

### 1.1 当前结构（v1）

```
世界列表 → 角色列表 → 对话会话
```

用户旅程：**先建角色，再聊天**。世界是角色的容器，没有独立价值。

### 1.2 目标结构（v2）

```
世界列表
└── 世界详情（三栏标签页）
    ├── 构建（Forge）   ← 新增
    ├── 故事（Story）   ← 原有对话/写作空间，合并为统一入口 + 双预设
    └── 状态（State）   ← 新增
```

用户旅程重构为三段：
1. **在 Forge 里建造世界** → 创建地点、势力、事件、关系，AI 辅助填充细节
2. **在 Story 里写故事** → 选择预设（角色扮演 / 叙事），带着完整世界背景开始创作
3. **在 State 里观察演化** → 看世界状态如何随故事推进而变化，设置触发规则

三段不强制串行——可以边写故事边补世界设定，随时切换。

**关于 Story 模式的会话统一**：v1 中"对话会话"和"写作会话"是两种独立入口，v2 合并为**一种"故事会话"**，差异收敛为会话属性 `preset`（见 §5）。

## §2 Forge 模式（世界构建工作台）

### 2.1 入口

世界详情页顶部有三个标签页：**构建 · 故事 · 状态**。默认进入上次停留的标签，首次进入世界默认进入构建。

### 2.2 界面布局

```
┌─────────────────────────────────────────────────────────────┐
│  [构建] [故事] [状态]          世界名称          [助手] [设置] │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│  实体    │              实体卡片区域                         │
│  类型树  │                                                  │
│          │  ┌─────────┐ ┌─────────┐ ┌─────────┐           │
│ ▶ 角色   │  │ 实体卡  │ │ 实体卡  │ │  + 新建 │           │
│ ▶ 地点   │  └─────────┘ └─────────┘ └─────────┘           │
│ ▶ 势力   │                                                  │
│ ▶ 事件   │  ← 切换到"关系"时主区域变为关系图/列表二合一视图   │
│ ▶ 关系   │                                                  │
│          │                                                  │
├──────────┴──────────────────────────────────────────────────┤
│  [写卡助手] 告诉我你想添加什么…                    [发送]    │
└─────────────────────────────────────────────────────────────┘
```

左侧：5 种实体类型的导航树，当前选中类型高亮。
主区域：所选实体类型的卡片网格（同当前角色列表样式）。
底部：写卡助手输入框常驻，随时可呼出 AI 辅助。

### 2.3 实体状态标签——固定预设 + 世界级自定义

状态标签（如"现存/已毁"）不是每个实体单独填写，而是**在世界设置里统一定义这个世界的标签集**，实体创建时从列表选。

每种实体类型有内置默认标签集，开箱即用；用户在世界设置的"标签管理"里可以增删改，改动影响整个世界所有同类实体。

**各类型默认标签**：

| 实体类型 | 默认状态标签 |
|---|---|
| 地点 | 现存 / 已毁 / 传说中 |
| 势力 | 活跃 / 解散 / 潜伏 |
| 事件 | 历史 / 进行中 / 预言 |

**事件可见性**（独立于状态标签，固定两选一，不可自定义）：`公开` / `秘密`
- 秘密事件：NPC/世界背景知道，但默认不注入玩家上下文（见 §2.6 注入规则）

### 2.4 五种实体类型详情

**角色（Character）** — 已有，无需变更数据模型，在 Forge 视图中与其他实体并列展示。

---

**地点（Location）**

卡片展示：名称 · 类型图标（城市/区域/建筑/自然地貌/其他）· 一行简介 · 状态标签 · 关联势力数量徽章 · 上级地点名（若有）

详情面板字段：
- 名称、类型（下拉）、状态标签（从世界标签集选）
- 上级地点（下拉关联已有地点，支持嵌套层级）
- 描述（富文本，同角色 system_prompt 编辑区样式）
- 关联势力（多选）
- 关联角色（只读反查：关系表中"长居/出生于此"的角色）
- 关联事件（只读反查：相关地点包含本地点的事件）
- **注入上下文**：开/关 toggle（见 §2.6）

---

**势力（Faction）**

卡片展示：名称 · 类型图标（王国/组织/宗教/商会/其他）· 一行简介 · 状态标签 · 领导者名 · 成员数量徽章

详情面板字段：
- 名称、类型（下拉）、状态标签
- 领导者（关联已有角色，可选）
- 描述
- 活跃地点（关联地点多选）
- 成员列表（关联角色多选，每个成员可附职位文本）
- 关联事件（只读反查）
- **注入上下文**：开/关 toggle

---

**事件（Event）**

事件是世界历史节点——不是对话内容，而是世界观层面发生过或将要发生的事情。

卡片展示：名称 · 世界内时间戳（纯文本，如"第三纪元 204 年春"）· 状态标签 · 可见性标签

详情面板字段：
- 名称、世界内时间戳（自由文本）
- 状态标签（从世界标签集选）
- 可见性：公开 / 秘密（固定两选）
- 描述（事件经过）
- 相关角色（多选）
- 相关势力（多选）
- 相关地点（多选）
- 影响描述（该事件对世界的持续性影响，可选）
- **注入上下文**：开/关 toggle；秘密事件此 toggle 额外附注"秘密事件默认关闭"

---

**关系（Relation）**

关系没有独立卡片列表，选中"关系"后主区域呈现两个并排视图，可切换。

**视图 A：关系图（Graph View）**
- 节点 = 各类实体，按类型着色：角色（朱砂）、势力（铁胆墨）、地点（褪色墨）、事件（灰）
- 连线粗细 = 关系强度（1-10），标签 = 关系类型
- 布局：圆形/层级静态布局（v2 不做力导向弹簧图）
- 支持按实体类型过滤（勾选显示哪些类型的节点）
- 点击连线弹出关系详情气泡

**视图 B：关系列表（List View）**
- 每行：实体A ↔ 关系类型 ↔ 实体B · 强度（数字）· 备注摘要
- 按实体筛选：选定某个实体后只显示与该实体相关的行
- 点击行内编辑

**新建/编辑关系**（弹窗）：
- 选择两端实体（跨类型均可，从下拉搜索选）
- 选择关系类型（预设 + 可自定义文本）：
  - 角色↔角色：盟友 / 敌对 / 恋人 / 家人 / 主从 / 师徒 / 竞争
  - 角色↔势力：成员 / 领袖 / 叛徒 / 追随者 / 敌人
  - 角色↔地点：出生地 / 长居 / 流亡于 / 统治
  - 势力↔势力：同盟 / 战争 / 附属 / 竞争
  - 势力↔地点：控制 / 驻扎 / 发源地
  - 其他组合：自由文本输入
- 强度（1-10 滑块）
- 备注（自由文本）

### 2.6 Forge 实体的上下文注入机制

这是 Forge 实体影响 AI 生成的核心机制。

**注入方式**：实体不走独立的 Prompt 条目列表，而是**以实体为单位整体注入**，内容 = 实体描述 + 自动附加的关系摘要。注入后的内容格式：

```
[地点·暮霞港]
类型：港口城市 · 状态：破败
描述：...
关系：暗影帮（控制），海商联盟（历史总部，已离开）
```

**关系随实体注入**（关键决策）：关系不单独作为 Prompt 条目管理，而是**附加在相关实体的描述尾部**一起注入。当某个实体被注入时，自动追加该实体强度最高的前 N 条关系（N 由常量控制）。这样关系跟着实体走，无需独立管理。

**注入预算制**（防止上下文爆炸）：
- 所有 Forge 实体注入共享一个 token 上限（全局常量，后续写入 `constants.js`）
- 注入优先级（高→低）：`当前场景激活角色` > `激活角色的关系` > `当前所在地点` > `当前场景相关势力` > `公开历史事件` > `其余实体`
- 超出预算时，低优先级实体自动截断；不影响用户手写的 Prompt 条目（两套配额分开）

**秘密事件不注入**：可见性为"秘密"的事件，即使"注入上下文"已开启，也不自动注入玩家侧上下文。只有通过触发器激活（§4.4）或用户在 Story 里 `@引用`（§5.1）时才注入。

**Prompt 条目层级收敛**（v2 决策）：
- 保留**世界级**（手写的世界特有知识，补充 Forge 实体未覆盖的内容）
- **废弃全局级 Prompt 条目**：跨世界的写作规范统一写入全局 system_prompt 文本框，不再维护独立的全局条目列表
- **废弃角色级 Prompt 条目**：角色相关的背景知识迁移到 Forge 实体描述 + 关系，通过实体注入机制处理
- 现有全局级/角色级条目数据保留，但 UI 不再提供创建入口

## §3 写卡助手升级（Forge 层全局助手）

### 3.1 当前能力（v1）

仅能操作角色卡：创建角色、修改状态字段、修改 Prompt 条目。

### 3.2 升级后能力（v2）

助手可操作所有五种实体类型，并理解实体之间的引用关系。

**示例对话**：

```
用户：帮我创建一个破败的港口城市，属于主世界，
      曾经是海商联盟的总部，现在被暗影帮控制。

助手：好的，我将创建以下内容：
      • 地点：暮霞港（类型：港口城市，状态：破败）
      • 关系：海商联盟 → 暮霞港（历史总部，状态：已离开）
      • 关系：暗影帮 → 暮霞港（控制，强度：9）
      确认后执行？[全部接受] [逐条确认]
```

```
用户：给我生成这个世界的三个贵族家族，
      互相之间有政治联姻但暗地里都想吞并对方。

助手：生成如下（6 项，可逐条接受/拒绝）：
      ✓ 势力：艾伦家族
      ✓ 势力：塔尔家族
      ✓ 势力：维斯家族
      ✓ 关系：艾伦↔塔尔（政治同盟，强度 5）
      ✓ 关系：塔尔↔维斯（政治同盟，强度 5）
      ✓ 关系：艾伦↔维斯（暗中敌对，强度 8）
      [全部接受] [逐条确认]
```

### 3.3 批量提案与逐条确认

Proposal 支持批量创建多个实体。预览卡列出每一项，用户可以：
- **全部接受**：一键执行所有项
- **逐条确认**：每项有独立的接受/拒绝按钮，未接受的项不执行
- 接受后实体立即出现在 Forge 卡片区

## §4 State 模式（世界状态引擎）

### 4.1 入口

点击顶部"状态"标签进入，上方是时间线，下方是触发器列表。

### 4.2 界面布局

```
┌─────────────────────────────────────────────────────────────┐
│  [构建] [故事] [状态]          世界名称          [助手] [设置] │
├─────────────────────────────────────────────────────────────┤
│  时间线区域                      会话选择下拉 ▼              │
│                                                              │
│  监控字段：[凛.好感度 ×] [世界.戒严等级 ×] [+ 添加字段]      │
│                                                              │
│  100 ┤  ╭──╮                                                │
│   80 ┤╭─╯  ╰──╮    ╭────                                   │
│   60 ┤╯        ╰──╯                                         │
│      └──┬──┬──┬──┬──┬── 轮次                                │
│         1  5  10 15 20                                       │
│                                                              │
│  ▲ 第12轮触发：表白剧情解锁（点击跳转）                       │
│                                                              │
├─────────────────────────────────────────────────────────────┤
│  触发器                                   [+ 新建触发器]     │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │ ● 启用                          上次触发：第12轮    │     │
│  │ 当 [凛.奥斯卡的信任度] [>] [50]                     │     │
│  │    且 [世界.戒严等级] [<] [4]                       │     │  ← 多条件AND
│  │ 则 激活条目 [公爵身份揭露线索]          [编辑] [删除]│     │
│  └────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 状态时间线

**字段选择器**：从当前世界的所有状态字段中多选（格式 `实体名.字段名`），每个字段一条折线，颜色区分。

**时间轴规则**：
- 横轴：对话轮次编号
- 纵轴：字段值
  - 数值字段：折线图
  - 文本字段：时间轴下方一排色块（相同值同色，值变化时换色），悬停显示文本
- 悬停数据点：tooltip 显示轮次号 + 字段值 + 该轮对话摘要
- 点击数据点：Story 模式的消息列表滚动定位到该轮次
- 触发器激活的轮次：时间轴上标记 ▲ 符号 + 触发器名称

**多会话支持**：顶部下拉选择当前世界下的某个故事会话，时间线按所选会话展示。

### 4.4 触发器

**触发器结构**：一个触发器 = 多条条件（AND 关系）+ 一个动作

```
条件1：[目标字段] [操作符] [值]
条件2：[目标字段] [操作符] [值]   ← 可添加多行，全部满足才触发
且...
────────────────────────────────
动作：[动作类型] [参数]
```

OR 逻辑的场景用**两个独立触发器**实现（每个触发器是一组 AND 条件），不引入条件组嵌套。

**条件目标格式**：`实体名.字段名`，例：`凛.好感度`、`世界.政局稳定性`、`玩家.声望`

**条件操作符**：
- 数值字段：`>` `<` `=` `>=` `<=` `!=`
- 文本字段：`包含` / `等于` / `不包含`

**动作类型**（三选一）：
1. **激活 Prompt 条目**：选择已有条目，触发后该条目自动进入激活状态（持续生效，直到用户手动关闭）
2. **注入系统提示**：填入一段文字，下一次生成时追加到系统 prompt 尾部（**仅生效一次**，生效后自动标记为已消耗）
3. **前端通知**：弹出通知消息，内容自定义（仅告知用户，不影响 AI）

**触发时机**：每轮 AI 回复完成、状态栏更新完毕后，自动检查所有启用中的触发器，满足条件立即执行动作。

**触发器卡片信息**：启用/禁用 toggle、条件摘要、动作摘要、上次触发轮次（若从未触发显示"从未触发"）。

**同一触发器的重复触发**：默认允许重复触发（每轮都检查）。用户可在触发器设置里切换"仅触发一次"（触发后自动禁用）。

## §5 Story 模式——双预设体系

### 5.1 统一入口与预设选择

v1 的"对话会话"和"写作会话"在 v2 合并为**故事会话**，差异收敛为一个会话属性：**预设（preset）**。

新建故事会话时，用户必须显式选择预设（无默认值，强迫有意识选择）：

```
新建故事会话
├── ◉ 角色扮演  —— 沉浸对话，角色以第一人称说话
└── ○ 叙事创作  —— 协作写作，AI 以作者视角推进故事
```

预设确定后可在会话设置里随时切换。切换只影响渲染层和 prompt，历史消息数据不变。

---

### 5.2 预设 A：角色扮演（Roleplay）

**定位**：沉浸式角色扮演，AI 化身角色用第一人称回应。对应 v1 对话会话。

**完整预设内容**：

| 层 | 具体内容 |
|---|---|
| UI 组件 | 气泡消息列表（头像 + 气泡，现有 ChatPage 样式） |
| CSS | 气泡圆角、头像、对话节奏间距；AI 气泡与用户气泡视觉区分 |
| Prompt 模板 | 第一人称角色扮演指令：`你正在扮演 {{char}}，用第一人称回应，保持角色语气` |
| 输出预期 | 短-中篇幅，对话节奏，不过度叙述动作 |
| 单角色行为 | AI 完全以该角色身份回应 |
| 多角色行为 | **SillyTavern 风格**：AI 在回复开头标注发言角色名，每次只以最合适的一个角色回应；用户可在输入框 `@角色名` 指定下一个发言角色 |
| 状态更新 | 角色状态栏 + 玩家状态栏（不更新世界状态栏） |

**多角色 Prompt 逻辑**（角色扮演预设专属）：

```
System：你同时扮演以下角色：[A]、[B]、[C]。
        每次只以最合适回应当前情景的一个角色身份说话。
        在回复第一行写"【角色名】"，然后以该角色第一人称继续。
        除非用户明确 @某个角色，否则你自主判断由谁发言。
```

### 5.3 预设 B：叙事创作（Narrative）

**定位**：协作写作，AI 作为作者推进叙事，第三人称描写动作与对白。对应 v1 写作空间。

**完整预设内容**：

| 层 | 具体内容 |
|---|---|
| UI 组件 | 章节段落块（无气泡，连续散文排版，现有 WritingSpace 样式） |
| CSS | 无气泡圆角；文字左对齐，行间距宽松；书页质感；段落间有分隔线 |
| Prompt 模板 | 第三人称叙事指令：`你是一位创意写作协作者，以第三人称推进故事，描写动作、环境和对白` |
| 输出预期 | 中-长篇幅，段落叙述，包含场景描写 |
| 单角色行为 | AI 以作者视角描写该角色的行动与内心 |
| 多角色行为 | **现有写作空间行为**：AI 统筹所有激活角色，一次性写出多角色的动作与对白 |
| 状态更新 | 角色状态栏 + 玩家状态栏 + 世界状态栏（全量更新） |

### 5.4 两套预设的共用功能

以下功能不因预设不同而变化：

**@ 世界知识引用**：输入框中输入 `@` 触发实体选择器弹窗，列出当前世界所有实体（角色/地点/势力/事件），按名称模糊搜索，选中后插入 `@实体名` 标记。发送时自动将该实体完整描述注入 AI 上下文（不显示在消息气泡/段落里），不受注入预算限制。秘密事件可通过 `@` 引用，这是手动注入秘密内容的唯一入口。

**会话侧边栏**：每个会话项显示预设徽章（`角色扮演` / `叙事`）+ 最后一条状态快照的前 2 个字段值预览。

**触发器激活标记**：某轮 AI 回复后有触发器激活，该轮消息底部显示小徽章 `◆ 触发：XXX`，点击跳转到 State 模式触发器详情。

## §7 功能边界（v2 明确不做）

| 功能 | 理由 |
|---|---|
| 多用户协作 | 单用户本地工具，v2 不引入多人实时协同 |
| 关系图力导向布局 | 实现复杂，数据量小时比静态布局更难看，v3 再评估 |
| 事件甘特图 | 世界时间线纯列表展示，不做可拖拽非线性编辑 |
| 触发器条件组（OR嵌套） | OR 场景用两个独立触发器实现，不引入条件组概念 |
| 章节/EPUB 导出 | 世界百科导出 Markdown 纯文本，不做排版 |
| 移动端适配 | 继续桌面端优先 |
| 角色级/全局级 Prompt 条目（UI入口）| 功能由 Forge 实体注入机制（角色级）和全局 system_prompt 文本（全局级）取代，数据保留但不新建 |
| 叙事分支（Branch）| 产品复杂度超出当前阶段，引入"平行世界"认知负担；回滚需求通过状态快照+重新生成覆盖 |
| 预设热切换时的消息重渲染 | 切换预设后历史消息保持原渲染样式，不回溯重渲（气泡变段落），从下一条消息开始生效 |

## §8 用户旅程完整示例

以一个 TRPG 爱好者为例：

```
1. 新建世界"深渊历纪"

2. [Forge] 构建世界骨架
   - 地点：帝都"阿尔卡城"· 古战场"血月平原"· 秘境"沉默图书馆"
   - 势力：帝国军 · 盗贼公会 · 秩序教会
   - 角色：主角"凛" · 反派"奥斯卡公爵" · 向导"老商人"
   - 关系：凛→帝国军（逃兵，强度9）· 凛→奥斯卡（追猎/被追猎，强度8）
   - 写卡助手批量生成 5 个帝国军配角

3. [世界设置] 配置状态字段模板
   - 凛：体力（0-100）· 秘术值（0-100）· 奥斯卡的信任度（-100~100）
   - 世界：帝国戒严等级（1-5）· 盗贼公会风评（文本）

4. [State] 配置触发器
   - 触发器A：当 凛.奥斯卡的信任度 > 50 且 世界.帝国戒严等级 < 4
              则 激活条目"公爵身份揭露线索"
   - 触发器B：当 世界.帝国戒严等级 >= 4
              则 注入系统提示"城市进入高度戒备，NPC 行为应更谨慎"（仅一次）

5. [Story] 开始写故事
   - AI 感知所有世界背景（地点/势力/关系随对话内容按需注入）
   - 每轮后自动更新状态（凛体力-10，奥斯卡信任度+5…）
   - 第 12 轮信任度突破 50，触发器A激活，AI 下轮自然引入揭露线索

6. [State] 回看演化
   - 时间线显示"奥斯卡信任度"从 -30 爬升到 +60 的弧线
   - 第 12 轮有 ▲ 标记，点击跳转到该轮对话

7. [Story] 探索另一走向
   - 在第 8 轮"关键抉择"处，从状态快照回滚到该轮状态
   - 重新生成，让凛背叛公会，观察世界向另一方向演化
```
