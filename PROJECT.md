## 是什么

WorldEngine 是一个本地运行的 Web 端 LLM 前端，专注于**角色扮演和创意写作**场景。

对标 SillyTavern，但做减法和优化：
- 删除使用频率低的功能（TTS、图生图、Visual Novel 模式等）
- 对核心功能深度优化（记忆系统、提示词系统、UI 交互）
- 牺牲部分可配置性，换取更好的开箱即用体验

## 核心设计理念

### 层级结构（比 ST 多一层"世界"）

```text
全局 → 世界 → 角色 → 会话
```

- **全局**：文风偏好、API 配置、默认提示词
- **世界**：世界观设定、规则、氛围——一个世界可以有多个角色
- **角色**：人设、背景——属于某个世界
- **会话**：具体的一次对话

**为什么加"世界"层**：ST 里同一个世界观下的多个角色，需要在每个角色卡里重复设定世界背景。WorldEngine 把世界背景提到世界层，角色只管角色自己的事。

### 提示词系统

两类提示词：
1. **System Prompt**：传统方式，全局/世界/角色三层合并注入
2. **Prompt 条目（自动触发）**：每轮注入所有条目的简介，代码根据 embedding 相似度和关键词匹配决定哪些条目展开正文。避免把所有内容塞进上下文，节省 token，保持 LLM 专注

OOC 内容（用户用 (( )) 标注的旁白说明）原文进入上下文，不做特殊过滤，由 AI 自行理解处理。
补充约束：
- OOC 参与当轮对话理解、会话标题生成和 Prompt 条目触发
- OOC 不应沉淀为角色长期状态或世界历史事实

### 记忆系统（核心创新）

**层一 - 原始 Session**：
- 全量保存，不丢失任何信息
- 每个 session 有 AI 生成的 summary
- 召回逻辑：embedding 搜索 → 发 summary → AI 决定展开 → 渐进式读取原文
- 类似"先看目录，再翻正文"

**层二 - 角色状态栏**：
- 每个世界可前端配置一套角色状态字段模板
- 每个角色基于该模板持有自己的当前状态值
- 用于记录角色的持续性个人状态，例如关系、伤势、心情、目标、身份等
- 后端在对话后按配置异步更新；对 LLM 注入时渲染为可读文本

**层二补充 - 玩家状态栏**：
- 每个世界可前端配置一套玩家状态字段模板（与角色状态字段模板独立）
- 每个世界有且仅有一个 persona 实例，持有一份当前值
- 玩家卡只有 name 与 system_prompt 两个基础字段，**不含 Prompt 条目**（与角色不同）
- 用于记录玩家的持续性个人状态，例如体力、背包、当前任务等
- 与角色状态栏同机制：后端在对话后按配置异步更新；对 LLM 注入时渲染为可读文本

**层三 - 世界状态栏**：
- 每个世界可前端配置一套世界状态字段模板
- 每个世界持有一份共享的当前世界状态
- 用于记录世界层的持续性公共状态，例如时间、局势、秩序、天气、地点状态、主线阶段等
- 后端在对话后按配置异步更新；对 LLM 注入时渲染为可读文本

**层四 - 世界时间线**：
- 记录这个世界里发生过的所有历史事件
- 按时间顺序，每事件一句话
- 给 AI 提供世界级别的历史背景

### 不做的功能

- 多用户系统
- 图片生成（SD、ComfyUI 等）
- TTS 语音
- Visual Novel 模式
- 多角色群聊（MVP 阶段）
- 云端同步
- 插件市场
- 复杂宏脚本系统
- SillyTavern PNG 角色卡兼容
- SillyTavern Regex 扩展格式兼容（仅实现同类能力，不导入 ST 正则包）
- 深浅色主题切换（外观完全交给用户自定义 CSS）
- 用户自定义 Prompt 组装顺序
- 自动备份（MVP 阶段不做）
- 消息分支 / Swipe（重新生成覆盖旧版本）
- 备选开场白（first_message 仅支持单条）
- Author's Note

## 用户体验原则

1. **开箱即用**：新用户创建一个角色就能开始聊，不需要理解复杂配置
2. **渐进披露**：高级功能存在但不强迫用户学习
3. **本地优先**：数据不出本机，API Key 只存本地
4. **简洁 UI**：参考 ST 的交互模式，但视觉上更简洁现代
5. **长期可用**：世界观、角色状态、历史关系可以持续积累

## 技术选型理由

| 选择 | 理由 |
|---|---|
| React + Vite | 组件化，适合模块化开发；Vite 构建快 |
| TailwindCSS | 不需要手写 CSS 文件，AI 协作友好 |
| Node.js + Express | 和前端同语言（JS），降低认知负担 |
| SQLite | 单文件数据库，无需安装服务，本地运行完美 |
| JSON 导出 | 数据可读，方便手动备份和迁移 |
