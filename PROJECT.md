# WorldEngine · 产品形态

> 本文件描述产品当前实际形态：用户看到什么、能做什么、体验是什么。
> 技术实现细节见 `ARCHITECTURE.md`，数据结构见 `SCHEMA.md`，执行约束见 `CLAUDE.md`。
> 最后更新：2026-04-25

---

## §0 产品定位

**WorldEngine 是一个 AI 辅助的沉浸式角色扮演与创意写作工具。**

核心用户画像：
- 有自己世界观设定的创意写作者和互动小说爱好者
- TRPG 玩家 / GM，需要管理多个角色、维护世界状态
- 想用 AI 辅助构建复杂角色系统与长程叙事的人

不面向的用户：只想快速开一个对话和角色聊天的人（SillyTavern 更适合他们）。

---

## §1 整体结构

### 1.1 导航层级

```
世界列表
└── 世界详情
    └── 角色列表
        └── 会话（对话 / 写作）
```

用户旅程：**先建世界 → 建角色 → 开会话**。世界是角色的命名空间，角色携带状态字段和提示词条目，会话在角色上下文中运行。

### 1.2 会话类型

| 类型 | 说明 |
|---|---|
| 对话（Chat） | 单角色对话，气泡消息列表，AI 以角色身份回应 |
| 写作（Writing） | 多角色协作写作，段落块排版，AI 统筹所有激活角色 |

两种类型在架构上共用一张 sessions 表，差异由 session 的 type 字段区分。

---

## §2 世界管理

### 2.1 世界列表页

卡片展示：封面图（可上传）、世界名称、一行描述。

### 2.2 世界编辑

字段：名称、描述、封面图、LLM 生成参数（模型、温度、上下文长度等，可覆盖全局配置）。

### 2.3 世界配置

世界配置页包含：
- **提示词条目**：当前世界的提示词条目列表，可设置触发类型、启用/禁用
- **正则替换规则**：世界级正则规则，作用于此世界的所有会话
- **自定义 CSS**：覆盖全局样式的世界级样式片段

### 2.4 世界状态字段

世界有独立的状态字段集（如"政局稳定性""戒严等级"），字段类型支持 text / number / boolean / enum / list。世界状态分为全局默认值和会话级运行时值——同一世界的不同会话状态相互独立。

---

## §3 角色管理

### 3.1 角色列表

卡片展示：头像、名称、一行简介。

### 3.2 角色编辑

字段：名称、简介、头像（文件上传）、系统提示词（富文本编辑）、生成参数（可覆盖世界/全局）。

### 3.3 角色状态字段

每个角色有独立的状态字段集（如"好感度""体力""当前情绪"），同样支持全局默认值与会话级运行时值分离。

---

## §4 提示词条目系统

提示词条目是驱动 AI 行为的核心机制，分为四种触发类型：

| 触发类型 | 说明 |
|---|---|
| `always` | 常驻激活，每轮必然注入 |
| `keyword` | 用户消息中出现关键词时触发 |
| `llm` | 由 AI 判断是否与当前上下文相关，按 token 权重排序注入 |
| `state` | 当状态字段满足条件表达式时自动激活 |

条目可以在世界级配置，三面板（玩家/角色/条目）可直观调整顺序与权重。

**状态条件（`state` 类型）**：条件目标格式 `实体名.字段名`，支持 `>` `<` `=` `>=` `<=` `!=`（数值），以及 `包含` / `等于` / `不包含`（文本）。

---

## §5 对话会话

### 5.1 界面

气泡消息列表，左侧侧边栏展示会话列表；右侧三个状态面板（玩家状态 / 角色状态 / 世界状态）始终可见，每轮 AI 回复后自动更新。

### 5.2 消息操作

- **重新生成**：替换最后一条 AI 消息
- **续写**：让 AI 接着上一条回复继续生成
- **模拟**：以角色视角生成一条额外的发言（不计入对话轮次）
- **编辑 / 删除**：支持消息内容编辑和删除（同步回滚状态）

### 5.3 图片附件

用户消息支持最多 3 张图片（单张不超过 5MB），base64 随消息发送，后端存储路径在 messages 表。

### 5.4 Token 消耗显示

每条 AI 回复可展示本轮 input / output / cache（若有）token 消耗，可在设置里开关。

---

## §6 写作会话

### 6.1 界面

段落块排版（无气泡），连续散文风格，章节自动分组，章节标题由 AI 生成。

### 6.2 多角色写作

写作会话可激活多个角色，AI 统筹所有激活角色，一次性写出多角色的动作与对白。

### 6.3 章节管理

每 N 轮（可配置）自动分为一章，章节标题异步生成后更新显示。

### 6.4 续写

写作会话同样支持"续写"操作，让 AI 接着上一段继续推进叙事。

---

## §7 状态系统（三层）

WorldEngine 维护三层状态，每层都有字段定义和运行时值：

| 状态层 | 挂载对象 | 典型字段示例 |
|---|---|---|
| 世界状态 | 世界 | 政局稳定性、戒严等级 |
| 角色状态 | 角色 | 好感度、体力、当前情绪 |
| 玩家状态 | Persona | 声望、持有道具 |

**状态更新模式**：
- `llm_auto`：每轮 AI 回复后，由 LLM 解析对话内容自动更新状态值
- `manual`：只能由用户手动编辑，AI 不自动修改

**会话级隔离**：每个会话有自己的一套状态快照，与其他会话互不干扰。

## §8 记忆系统

### 8.1 Turn Record（逐轮摘要）

每轮 AI 回复完成后，异步生成 10–50 字的摘要，写入 turn_records 表并同步生成向量 embedding。

### 8.2 向量相似度召回

新消息发送时，系统检索历史 turn records，将语义相关的历史片段注入提示词。同 session 相似度阈值 0.72，跨 session 阈值 0.84。

### 8.3 智能展开

决策引擎判断某段摘要是否需要注入原文（而非仅摘要），在上下文预算允许时展开完整内容。

---

## §9 日记系统

AI 自动为每个角色/玩家生成日记条目，有两种时间模式：

| 模式 | 说明 |
|---|---|
| 真实日期 | 按现实日历，当日对话结束后生成当天日记 |
| 虚拟日期 | 按世界内时间戳，日期跨越时触发生成 |

日记作为长程记忆注入提示词（见 `ARCHITECTURE.md §4`，第 [10] 段）。

---

## §10 写卡助手

### 10.1 定位

写卡助手是一个挂载在界面右侧/底部的 AI 代理面板，专门辅助用户构建和编辑世界内的各类配置内容，而非参与对话叙事。

### 10.2 当前能力

助手支持以下 Agent Skill：

| Skill | 能力 |
|---|---|
| 角色卡 | 创建角色、编辑系统提示词、修改状态字段、增删提示词条目 |
| 世界卡 | 创建/编辑世界、添加世界状态字段、管理提示词条目 |
| 玩家卡 | 创建/编辑 Persona、设置玩家状态字段 |
| 全局配置 | 修改全局提示词、全局系统提示词 |
| 自定义 CSS | 新增/修改自定义样式片段 |
| 正则规则 | 创建/编辑正则替换规则 |

### 10.3 提案机制

助手不直接执行写操作，而是生成**提案（Proposal）**，用户在预览卡上确认后方执行。支持逐条接受/拒绝。提案执行过程通过 SSE 实时推送进度。

---

## §11 正则替换与自定义 CSS

### 11.1 正则替换

规则按 scope 划分为四类：

| Scope | 作用时机 |
|---|---|
| `user_input` | 用户消息发送前处理 |
| `ai_output` | AI 回复写入数据库前处理 |
| `display_only` | 仅影响前端展示，不入库 |
| `prompt_only` | 仅在提示词组装时处理，不影响消息内容 |

规则可设为仅在对话模式或写作模式下生效，或两者都生效。

### 11.2 自定义 CSS

CSS 片段可在全局设置或世界配置中添加，所有 `enabled` 的片段按顺序拼接后注入 `<style id="we-custom-css">`，全局作用。

---

## §12 Persona（玩家身份）

Persona 是用户在对话中扮演的"玩家角色"，有名称和系统提示词（user_prompt）。Persona 不同于世界中的 NPC 角色，没有提示词条目，但有独立的状态字段集。

---

## §13 LLM 配置

### 13.1 支持的提供商

| 提供商 | 说明 |
|---|---|
| Anthropic | Claude 系列，支持缓存 token 统计 |
| OpenAI | GPT 系列 |
| OpenAI 兼容 | DeepSeek、SiliconFlow 等第三方接口 |
| Google Gemini | Gemini 系列 |
| Ollama | 本地部署模型 |

### 13.2 生成参数覆盖层级

`世界级参数 > 全局参数`，世界级字段为 NULL 时回退全局配置。

可配置参数：模型选择、温度、top_p、max_tokens、频率惩罚、存在惩罚、上下文消息条数等。

### 13.3 模型价格显示

模型下拉中直接展示每百万 token 的 input / output / cache_write / cache_read 价格，方便成本预估。

## §14 导入导出

| 格式 | 说明 |
|---|---|
| `.wechar.json` | 单角色导出，含状态字段定义和状态值，format: worldengine-character-v1 |
| `.weworld.json` | 完整世界导出，含世界、所有角色、配置、会话历史，format: worldengine-world-v1 |
| `.weglobal.json` | 全局设置导出，含全局提示词、自定义 CSS、全局正则规则和非 LLM 配置，format: worldengine-global-settings-v1 |

全局设置导入为覆盖模式（先清空相关全局记录再写入），不含 API 密钥。

---

## §15 Electron 桌面应用

支持 macOS 和 Windows 打包，主要特性：

- 数据目录用户化（`~/Library/Application Support/WorldEngine` on macOS，`%APPDATA%\WorldEngine` on Windows），不写入应用包内
- 随机端口避免与本机其他服务冲突
- 打包命令：`npm run desktop:dist`（根目录）

---

## §16 全局设置

全局设置页包含：

- **LLM 配置**：默认提供商、模型、API Key、基础参数
- **全局提示词**：系统级提示词（注入所有会话最高层）、后置提示词、全局常驻条目
- **功能开关**：Token 消耗显示、自动生成标题等
- **正则规则**：全局作用的正则替换规则
- **自定义 CSS**：全局样式片段
