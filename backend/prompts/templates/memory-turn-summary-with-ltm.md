part2：长期记忆

长期记忆不是剧情日志，而是“当前活跃约束索引”。状态记忆另行维护，本模块只记录未来仍生效的因果、条件、联动、秘密、期限、禁忌和任务约束。

默认输出：无。

仅当本轮信息满足“新 + 活跃 + 长期 + 有明确锚点 + 会约束未来 + 不能被状态记忆替代 + 值得占用有限槽位”时，才输出更新。

类型：
规则 / 任务 / 人物 / 关系 / 势力 / 地点 / 物品 / 秘密 / 契约 / 禁忌

保留级别：
P5核心约束：必须遵守；违反会造成世界观、主线、硬规则、死亡/毁灭、身份、契约等重大矛盾。
P4强约束：高度影响人物行动、关系走向、任务成败、阵营冲突或关键物品使用。
P3活跃伏笔：当前仍有效，未来可能影响选择、揭示、风险、误会或冲突。
P2弱参考：可由状态记忆/摘要/上下文替代，不写进长期记忆。
P1废弃：不记录。

长期记忆只允许输出 P3/P4/P5。

时效：
永久 / 持续 / 直到X / X之前 / 未公开前 / 条件解除前 / 任务完成前

内容要求：
- 每行一条。
- 写当前仍生效的约束，不写剧情流水。
- 每轮最多新增1行。
- 无明确后续约束则不记录。
- 记录固定格式的前缀：[类型|保留级别|锚点|时效]
- 不记录临时情绪、动作、寒暄、气氛、重复设定、单纯事件、无后续影响的铺垫。
- 没有在本轮剧情中发生改变的世界观，不得记录。

────────────────────────────────────────
输出格式（严格 JSON，不要加任何额外说明、不要 markdown 围栏、不要思考过程）：

```
{"summary":"<摘要正文，10–30字>","memory":["<[类型|保留级别|锚点|时效] 内容>"]}
```

- `summary` 必填；不允许空字符串
- `memory` 是数组，最多 1 条，每条 10–20 字一句话，知识条目风格，不加序号、不加分类前缀
- 若没有任何条目满足全部要求，输出 `"memory":[]`，**禁止**编凑
