# 规则
1. 只输出一个 JSON 数组，每个元素是一个操作。除 JSON 外不输出任何字符。
2. 操作只有四种：
   - `{"table":"<表key>","op":"add","row":{列:值,...}}` 新建行（不要写 id，系统自动分配）
   - `{"table":"<表key>","op":"update","id":<行id>,"fields":{列:值,...}}` 改已有行（id 必须照抄上方表中真实存在的 id）
   - `{"table":"<表key>","op":"close","id":<行id>,"reason":"..."}` 归档退场的行（剧情线关闭、NPC 死亡、物品消耗），reason 会被记进归档
3. 只用上方「表结构」列出的 key 和列名，写错的列会被丢弃。
4. **新建前必须先查重**，避免重复行：
   - relations / world 这种没有单一主名的表：按全部关键列判重（relations 用「主体A+主体B+关系类型」，且 A、B 按字典序排列固定方向；world 用「规则/事实」）。
   - 其余表：按首列实体名和「别名」列判重。
   - 若实体已存在（哪怕换了称呼），用 update 改它并把新称呼追加进「别名」字段，不要新建重复行。
5. **写对表、对准主语**：每张表的「用途」说明了它的记录视角。地点表只记跟这个地方绑定的客观事实（归属、物理状态、危险/资源），主语永远是地点；角色之间的行动、阴谋进展、谁对谁做了什么属于剧情线表或关系表，不要把剧情流水写进地点表的「当前状态」「已发生事件」。世界状态表只记开局设定之后被剧情新打破/新造成的世界级变化，开局世界观设定里本就有、且没变化的静态前提不要登记。
6. 字段值精炼：只写结果不写过程，**每个字段 ≤60 字**（超出会被截断），「最近变化」「已发生事件」等控制在一句话内、名词化标记，不要叙事。
7. 只有发生**实质变化**才动表；闲聊、无信息推进时不输出任何操作（空数组 `[]` 即可，不需要为没动的表写 noop）。
8. **行数上限（见上方「各表行数上限」）**：某张表标「已满」或新增后会超过上限时，每要 `add` N 行，必须先对这张表 `close` 掉 N 行**最不重要**的旧行（最久没被提及、已无剧情价值、过期或已被取代的），再 add。务必腾够空间，让该表新增后不超过上限。不限制的表无需归档。归档优先级由你判断，不要机械地删最早那行。
